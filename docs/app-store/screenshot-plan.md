# Turn Timer Screenshot Plan

Date: 2026-06-27
Status: Draft storyboard

## Default Product Page

Use the default App Store product page to sell the clearest wedge first:
fair turns and reusable visual sequences. The first two screenshots need to work
even when seen as small thumbnails in search.

| # | Caption | Visual | Capture Notes |
|---|---|---|---|
| 1 | Keep game night moving | Active Game Night session with a large colorful timer, current player, and next player visible | Lead with the strongest visual. Use a realistic player list and a nearly half-full countdown. |
| 2 | See every turn | Round sequence view showing names, colors, repeats, skip/restart/do-over controls | Make the app feel purpose-built for turn order, not a generic countdown. |
| 3 | Save reusable templates | Templates tab with Game Night, Recipe Steps, Plant Watering, Classroom Stations, and Meeting Agenda | Show breadth after the game-night hook. |
| 4 | Reuse, share, and sync | Saved template library plus Pro affordance for iCloud sync/import/export/history | Position Pro as convenience and portability. Avoid price text in the screenshot. |
| 5 | Start from any surface | Favorite-template widget and Apple Watch timer shown with the iPhone app | Use only if widget/watch visuals are polished in the build used for screenshots. |

Caption rules:

- Keep captions under eight words.
- Use sentence case.
- Match the actual visual exactly.
- Avoid feature lists in the caption.

## Visual Direction

- Use straight-on device frames or full-screen captures. The app has enough
  actual color in the timer and template UI; do not hide it behind decorative
  backgrounds.
- Favor bright timer states, real template names, and legible round labels.
- Keep caption text large enough for search thumbnails.
- Avoid generic stopwatch imagery. The differentiator is named, reusable rounds.
- Use the same template data across screenshots so the story feels coherent.
